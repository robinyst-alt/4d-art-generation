## 1. 产品概述与愿景

### 产品定位

4D Art 是一个创意生成工具，让用户通过 AI 生成独特的四维像素艺术作品。用户可以自由探索四维物体的不同切片，感受超越三维的视觉体验。

### 核心价值主张

- **突破维度限制** — 将抽象的四维概念转化为可交互的视觉体验
- **创意与技术的结合** — AI 生成 + 实时可视化，让艺术创作无边界
- **独一无二的所有权** — Content Hash 确保每件作品的唯一性

### 目标用户

- 数字艺术爱好者 — 追求独特、新颖的艺术形式
- 技术探索者 — 对四维几何、维度理论感兴趣
- NFT 收藏家 — 寻求稀缺性强的数字艺术品
- 创作者 — 希望生成独特的 4D 艺术用于个人或商业用途

### 成功愿景 (Phase 1 Goal)

在第一阶段，用户可以快速生成一个数学定义的 4D 物体，通过四象限切片控制探索不同切面视角，感受四维空间的神秘与美。整个过程无需安装任何插件，打开浏览器即可体验。

**Phase 1 已实现状态**：
- 6 种预设 4D 物体：Tesseract、4D Sphere、4D Octahedron、4D Dodecahedron、4D Icosahedron、4D Torus
- 四象限切片面板 UI（xyzw 四个独立控制）
- 默认 w=12 唯一切片，xyz 自由轴
- 像素点密度控制（6 档：1-6）
- 3D 视角可自由旋转、缩放

- 坐标轴指示器（显示相机轴 XYZW，随相机视角同步旋转）

- 切片值可点击编辑（0-23）

## 2. 用户故事与使用流程

### 用户故事

#### US-001: 生成 4D 艺术作品 (P0)
作为用户，我希望通过简单的操作生成 4D 艺术作品，以便快速体验四维视觉的魅力
- 选择预设的四维物体（超立方体、4D 球体等）
- 点击生成，系统计算并渲染 4D 数据
- 获得唯一的 Content Hash 作为作品标识

#### US-002: 探索四维切片 (P0)
作为用户，我希望通过切片控制来探索四维物体不同位置的形态，以便深入理解四维结构
- 至少选择 1 个维度作为切片，切片值决定在该维度上的切面位置
- 剩余未选为切片的维度可以自由旋转/缩放，但移动其滑条不影响图形展示
- 移动切片值会改变 3D 图形的形态（图形变形）
- 例：w=12 切片 → XYZ 自由旋转；w=20 切片 → XYZ 图形变形

#### US-003: 多切片探索 (P0)
作为用户，我希望同时选择多个维度作为切片，以便观察更低维度的截面形态
- 可以选择 1-3 个维度作为切片
- 切片数量 = 4 - 剩余自由轴数
- 例：w=20 和 y=7 都是切片 → 展示 XZ 平面（可自由旋转）

#### US-004: 调整像素点密度 (P0)
作为用户，我希望调整像素点的疏密程度，以便获得不同的视觉效果
- 选择密度等级 1-6
- Level 1 = 默认密度，Level 2-6 = 递增的稀疏程度（2x, 3x, 4x, 5x, 6x）

#### US-005: 提交作品参加展示 (P1)
作为用户，我希望将我满意的 4D 作品提交到前端展示区，以便分享给更多人欣赏

### Phase 1: 数学定义物体 · 纯前端实现

#### F-101: 4D 物体选择器 (P0)
用户可从预设的四维物体列表中选择要生成的对象

| 字段 | 值 |
|------|---|
| 输入 | 单选下拉菜单 |
| 选项 | 超立方体, 4D 球体, 4D 八面体, 4D 十二面体, 4D 二十面体, 4D 环面 |
| 默认值 | 超立方体 |

#### F-102: 截图按钮 (P0)
一键将当前 3D 视角截图并自动保存到本地

| 字段 | 值 |
|------|---|
| 截图格式 | PNG |
| 保存方式 | 点击按钮自动下载到本地 |
| 文件命名 | `4d-art-{timestamp}.png` |

#### F-103: 4D 数据生成器 (P0)
根据选定的物体类型和参数，在前端计算生成 4D 像素矩阵

| 字段 | 值 |
|------|---|
| 输出格式 | Float32Array [w][z][y][x][rgba] |
| 数据量 | 24³ = 13,824 点 ≈ 50KB |
| 计算时间 | < 2s |

#### F-104: 四象限切片控制 (P0)
管理 xyzw 四个维度的切片/自由状态，实时探索四维结构

| 字段 | 值 |
|------|---|
| 象限数量 | 4 个 (x, y, z, w) |
| 模式 | 切片 / 自由 (每个轴独立选择) |
| 切片数量 | 至少 1 个，最多 3 个 |
| 切片值范围 | [0, 23]，整数 |
| 默认状态 | w=12 作为唯一切片，xyz 为自由轴 |
| 帧率 | Target 60fps，16ms 内完成切片提取 |

**切片逻辑**：
- **切片**：该维度作为切片，其值决定切面位置
  - 移动切片滑条 → 图形形态改变（变形）
- **自由**：该维度不作为切片，可以旋转/缩放视图
  - 移动自由轴滑条 → 不影响图形展示，仅改变视角

**场景示例**：

| 切片轴 | 切片值 | 自由轴 | 3D 视图 |
|--------|-------|--------|---------|
| w | 12 | xyz | XYZ 自由旋转（图形不变形） |
| w | 20 | xyz | XYZ 图形变形 |
| w, y | 12, 7 | xz | XZ 平面（可自由旋转） |
| w, y, z | 12, 7, 15 | x | X 轴线条 |

**UI 组件**：
- 四轴控制面板，每个轴包含：轴标签、切片/自由切换按钮、滑条、数值显示
- 切片/自由切换：`[切片]` 或 `[自由]` 按钮
- 约束：至少 1 个切片轴
- **数值显示可点击编辑**：点击数值显示区域可直接输入 0-23 的整数，支持 Enter 确认、Escape 取消、Tab 切换到下一轴

#### F-105: Three.js 3D 渲染器 (P0)
将当前切片数据渲染为可交互的视觉画面

| 字段 | 值 |
|------|---|
| 渲染引擎 | Three.js r158+, WebGL2, OrbitControls |
| 相机 | PerspectiveCamera，FOV 50° |
| 控制 | OrbitControls |
| 几何体 | Points / BoxGeometry 颗粒化 |
| 动态轴 | 根据固定状态动态计算参与渲染的轴 |

#### F-106: Content Hash 生成 (P1)
为每个生成的作品生成唯一的 Content Hash

| 字段 | 值 |
|------|---|
| 算法 | SHA-256，前 12 字符 |
| 格式 | 如: `a7f3b2c9d8e1` |

#### F-107: 像素点密度控制 (P0)
调整像素点之间的距离，使画面变得紧凑或稀疏

| 字段 | 值 |
|------|------|
| 控制方式 | 独立面板，1 个轴（P轴） |
| 密度等级 | 1, 2, 3, 4, 5, 6（共 6 档） |
| Level 1 | 默认间距（无缩放） |
| Level 2-6 | 默认距离的倍数（2x, 3x, 4x, 5x, 6x） |

**UI 组件**：
- 独立面板结构，类似 F-104 四象限控制
- 包含：轴标签（P）、滑条、数值显示
- 滑条范围：[1, 6]
- **数值显示可点击编辑**：点击数值显示区域可直接输入 1-6 的整数，支持 Enter 确认、Escape 取消

**状态定义**：
- `pointSpacing`: number — 密度等级 [1-6]
- 默认值: 1

#### F-108: 坐标轴指示器 (P1)
在左上角显示一个可折叠的小窗口，展示相机视角的坐标轴

**相机视角与坐标轴**：
- 坐标轴指示器显示的是**当前相机视角的坐标轴**
- 显示数量取决于相机轴数量（= 4 - 锁定数量）：
  - 锁定 = 0 → 相机轴 = 4 → 显示 XYZW 四轴（3D 旋转）
  - 锁定 = 1 → 相机轴 = 3 → 显示 XYZ 三轴（3D 旋转）
  - 锁定 = 2 → 相机轴 = 2 → 显示 XY 两轴（2D 平面观察，无旋转）
  - 锁定 = 3 → 相机轴 = 1 → 显示单轴（1D 沿线观察，无旋转）

**坐标轴显示规则**：
- 根据相机锁定状态动态显示轴
- 仅显示**相机轴**（非锁定的轴）
- 切片轴（锁定）不显示在坐标轴指示器中

**视觉样式**：
- 纯色/灰度（非彩色）
- 简单线条 + 轴标签
- 坐标轴随相机视角同步旋转

**UI 结构**：
```
┌─────────────────┐
│ [−]             │  ← 折叠按钮
├─────────────────┤
│     Y           │
│     ↑           │
│     │           │
│     └──→ X      │  ← 相机坐标轴（根据锁定状态动态）
│    /            │
│   Z             │

```

**状态定义**：
- `axisIndicatorVisible`: boolean — 是否显示坐标轴指示器
- 默认值: true（展开）

**新增功能（F-108 扩展）**：

1. **可折叠显示**

#### F-109: 相机锁定功能 (P1)
控制相机视角的维度，限制相机只能在某些轴的方向上观察图形

**术语定义**：
- **切片轴**：对图形切一刀的轴，决定图形截面
- **自由轴**：不决定图形截面的轴
- **相机锁定轴**：不参与相机视角的轴（必须是切片轴的子集）
- **相机轴**：参与相机视角的轴 = 非锁定轴

**关系规则**：
1. 相机锁定轴 ⊆ 切片轴（锁定的一定是切片，但切片的不一定是锁定）
2. 相机锁定轴 + 相机轴 = 4个轴
3. 切片轴 + 自由轴 = 4个轴

**切片轴含义**：
- 1 个切片 = 切 1 刀 → 展示 3D 图形在某平面的投影
- 2 个切片 = 切 2 刀 → 展示 2D 截面
- 3 个切片 = 切 3 刀 → 展示 1D 线/点

**相机旋转维度**：
| 锁定数量 | 相机轴 | 相机视角 | 旋转能力 |
|---------|--------|---------|---------|
| 0 | 4 (XYZW) | 4D | 3D 旋转 |
| 1 | 3 | 3D | 3D 旋转 |
| 2 | 2 | 2D | 2D 平面观察（无旋转） |
| 3 | 1 | 1D | 沿线观察（无旋转） |

**约束**：
- 切片轴：至少 1 个，最多 3 个
- 相机锁定轴：至少 1 个，最多 3 个
- 相机锁定轴 ⊆ 切片轴（强制约束）

**UI 组件**：
每个轴的控制行包含两个功能按钮（并排）：
- 切片/自由切换：`[切片]` 或 `[自由]` 按钮
- 锁定/解锁：`[🔒]` 或 `[○]` 图标按钮

**锁定按钮交互**：
- 只有"切片"模式的轴才能"锁定"（点击锁定时，如果当前是自由模式，自动转为切片）
- 锁定后该轴自动变为切片模式
- 取消锁定后该轴可自由选择切片/自由模式
- 至少保留 1 个锁定轴

**场景示例**：

| 切片轴 | 锁定轴 | 相机轴 | 相机视角 | 图形截面 |
|--------|-------|--------|---------|---------|
| W | W | XYZ | 3D | W 切面（YZ 投影） |
| W, X | W | XYZ | 3D | W∩X（YZ 平面） |
| W, X | W, X | YZ | 2D | W∩X∩Y（Z 线） |
| W, X, Y | W, X | YZ | 2D | W∩X∩Y（Z 线） |

**状态定义**：
- `lockedAxes`: string[] — 被锁定的轴列表（子集 of 切片轴）
- 默认值: ['w']

**实现文件**：
- `js/quadrant/controls.js` - 锁定按钮逻辑
- `js/quadrant/stateManager.js` - 锁定状态管理
- `js/render/camera.js` - 相机旋转维度限制

#### F-110: 渐变色彩系统 (P1)
为所有 4D 图形添加从中心到表面的渐变色彩

**色彩规则**：
1. 同一个像素只能是一种颜色
2. 从深灰色到白色，选取 8 种过渡色（固定色值）
3. 内部颜色深，外部颜色浅，整体深色到浅色渐变

**8 阶灰度色板**（调整后，避免白色过多）：
| 层级 | 颜色 | RGB 值 | 归一化值 |
|------|------|--------|----------|
| 1 (最内) | 深灰黑 | RGB(20, 20, 20) / #141414 | (0.08, 0.08, 0.08) |
| 2 | 深灰 | RGB(51, 51, 51) / #333333 | (0.20, 0.20, 0.20) |
| 3 | 暗灰 | RGB(97, 97, 97) / #616161 | (0.38, 0.38, 0.38) |
| 4 | 中灰 | RGB(140, 140, 140) / #8C8C8C | (0.55, 0.55, 0.55) |
| 5 | 浅灰 | RGB(179, 179, 179) / #B3B3B3 | (0.70, 0.70, 0.70) |
| 6 | 亮灰 | RGB(209, 209, 209) / #D1D1D1 | (0.82, 0.82, 0.82) |
| 7 | 浅灰白 | RGB(230, 230, 230) / #E6E6E6 | (0.90, 0.90, 0.90) |
| 8 (最外) | 近白 | RGB(245, 245, 245) / #F5F5F5 | (0.96, 0.96, 0.96) |

**渐变映射逻辑**：
- 计算像素到形状中心的标准化距离 [0, 1]
- 使用平方根曲线 (sqrt(dist)) 使颜色分布更均匀
- 中心点 = 最深色，边缘点 = 最浅色（不到达纯白）

**各图形渐变计算方式**：
| 图形 | 距离计算方式 |
|------|-------------|
| Tesseract | max(\|x\|, \|y\|, \|z\|, \|w\|) / halfSize |
| Sphere | sqrt(x²+y²+z²+w²) / radius |
| Octahedron | (\|x\|+\|y\|+\|z\|+\|w\|) / 0.9 |
| Dodecahedron | 3D 距离 / (0.9 × 1.5) |
| Icosahedron | 3D 距离 / (1.0 × 1.5) |
| Torus | tube 内距离 / minorRadius |

**状态定义**：
- 无新增状态（颜色由图形几何自动计算）

**实现文件**：
- `js/fourD/generators.js` - `GRADIENT_COLORS_8`, `getGradientColor()`, `distance3D()`, 各图形生成器中的渐变逻辑

## 4. 坐标系定义

### 坐标空间
- **空间范围**：[0, 23]^4（索引 0-23）
- **物体所占空间**：[1, 23]^4（实际物体范围 1-22）
- **物体中心**：[12, 12, 12, 12]
- **切片值范围**：[0, 23]

### 归一化
物体空间归一化到 [-1, 1]：
- `normalized = (index - 12) / 11`
- 索引 1 → -1.0
- 索引 12 → 0.0（中心）
- 索引 23 → 1.0

## 8. 附录：关键技术决策记录

### K-01: Content Hash vs 链上 NFT
Phase 1-2 使用 Content Hash（类似 Git commit），不上链。降低用户门槛，后续可升级绑定区块链。

### K-02: 切片/自由模式设计
切片决定切面位置，自由决定视角。概念清晰，交互直观。

### K-03: 像素点密度控制
通过缩放点坐标位置实现，无需修改几何体结构，实现简单高效。

### K-04: 中英文界面切换
使用语言切换按钮在中文/英文之间切换，所有 UI 文本通过 i18n 系统管理，提升国际化体验。
