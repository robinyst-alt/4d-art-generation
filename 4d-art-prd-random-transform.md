# PRD: 随机变幻 (Random Transform) 面板

**版本**: v2.1
**日期**: 2026-05-28
**状态**: 待评审

---

## 1. 概述与目标

### 1.1 功能概述

在 4D Art 应用中添加「随机变幻」面板，提供 3D 和 4D 随机旋转变换功能，允许用户对四维几何体进行随机角度旋转，并以新坐标系进行切片观测。

### 1.2 核心目标

- **3D变幻**：切换按钮，点击后在「正常模式」与「3D旋转模式」间切换，图形与自由轴的绑定关系改变
- **4D随机变幻**：对整个 4D 矩阵应用四维复合旋转矩阵，重采样回整数网格，动画时长 3-5 秒
- **复位**：一键还原到初始状态

## 3. 功能详情

### 3.1 3D 变幻（模式切换）

**前置条件**：无（始终可用）

**核心问题**：超立方体旋转后，如果切片仍沿原始轴方向，切出的永远是标准切面（正方形），无法观察到"真正的"旋转效果。

**解决思路**：让坐标轴固定在屏幕上（绑定相机视角），而不是随图形旋转。这样切片沿着屏幕坐标轴切，就能切出六边形等非标准形状。

---

#### 模式A：正常模式（默认）

```
图示：

    相机视角
       ↑
    ┌─────────────────┐
    │                 │
    │    图形 ───→    │  ← 图形旋转时，坐标轴跟着一起旋转
    │      ↑         │  ← 轴与图形保持相对静止
    │    坐标轴       │
    └─────────────────┘

特点：
- 图形绑定了3个自由轴（相对静止）
- 相机视角与坐标轴解绑（可自由拖拽旋转视角）
- 切片按固定坐标值提取（锁定轴的值由滑块控制）
- 自由轴由切片轴决定：切X则自由轴=YZW，切Y则自由轴=XZW，切Z则自由轴=XYW，切W则自由轴=XYZ
```

#### 模式B：3D旋转模式

```
图示：

    相机视角（固定不动）
       ↑
    ┌─────────────────┐
    │                 │
    │      图形旋转了  │  ← 图形相对于坐标轴有旋转变换
    │                 │
    │    ───→         │  
    │  坐标轴（绑定屏幕）│  ← 坐标轴始终朝上，不随图形旋转
    └─────────────────┘

特点：
- 图形与3个自由轴解绑（图形相对坐标轴有旋转变换）
- 拖拽图形时，相机视角固定不动，视觉上坐标轴固定不动
- 轴指示器不旋转（始终朝上）
- 切片按旋转后的坐标提取
- 自由轴关系不变
```

---

**实现方式（与模式A的区别）**：

| 方面 | 正常模式 | 3D旋转模式 |
|------|---------|-----------|
| 图形渲染 | 正常旋转显示 | 正常旋转显示（无额外计算） |
| 坐标轴追踪 | 无 | **后台实时记录R矩阵** |
| 切片计算 | 直接按滑块值提取 | **滑块值×新轴方向→三线性插值** |

**旋转矩阵**：
- 记录旋转矩阵 `R`（3x3，正交矩阵）
- 正常模式：`R = I`（单位矩阵）
- 模式B：拖拽时实时更新R

**切片计算公式**：
```
设用户滑块值为 (sliceX, sliceY, sliceZ)

新X轴方向 = R × (1, 0, 0)
新Y轴方向 = R × (0, 1, 0)
新Z轴方向 = R × (0, 0, 1)

实际采样点 P = sliceX × 新X轴 + sliceY × 新Y轴 + sliceZ × 新Z轴
            = R × (sliceX, sliceY, sliceZ)

从原矩阵用三线性插值提取点P
```

**示例**（X轴绕Z轴旋转45°）：
```
R = [[cos45, -sin45, 0],
     [sin45,  cos45, 0],
     [0,      0,     1]]

新X轴方向 = R × (1,0,0) = (cos45, sin45, 0) = (0.707, 0.707, 0)

用户 x=12 → 实际采样点 = 12 × (0.707, 0.707, 0) = (8.485, 8.485, 0)
```

**UI表现**：
- 按钮按下（模式B）：蓝色背景、白色字体
- 按钮松开（模式A）：白色背景、黑色字体
- 切换即时生效，无动画

**3D旋转生成**：

球坐标用于记录旋转角度，好处是能自然保持：
- 归一性（方向向量长度始终=1）
- 正交性（任意两轴点积=0）
- 连续性（角度平滑变化）

```
1. 生成球面坐标
   - theta ∈ [0, 2π)    // 绕 Y 轴方位角
   - phi ∈ [0, π]        // 与 Y 轴极角
   - 使用 Marsaglia 方法保证球面均匀分布

2. 构建旋转矩阵 R（绕Y轴旋转theta，绕X轴旋转phi）
```

**坐标轴方向计算**：

自由轴基向量固定为 (1,0,0)、(0,1,0)、(0,0,1)（由Phase 1座位分配机制决定分配给哪三个轴）。

图形旋转后，基向量随之旋转：

```
设旋转矩阵为 R

原基向量 (1,0,0) → 新方向 R × (1,0,0)
原基向量 (0,1,0) → 新方向 R × (0,1,0)
原基向量 (0,0,1) → 新方向 R × (0,0,1)
```

切片时，滑块值乘以旋转后的基向量得到实际采样点：

```
用户设置 x=12，y=0，z=0

若 R × (1,0,0) = (0.707, 0.707, 0)

采样点 P = 12 × (0.707, 0.707, 0) = (8.485, 8.485, 0)
```

### 3.2 4D 随机变幻

**前置条件**：无

**算法**：

```
1. 生成 6 个随机旋转角
   - θxy, θxz, θxw, θyz, θyw, θzw ∈ [0, π]

2. 构建 4D 旋转矩阵 R = Rxy * Rxz * Rxw * Ryz * Ryw * Rzw
   // 每个平面旋转矩阵均为 4×4 正交矩阵

3. 逐帧应用旋转并重采样
   for each grid point (w, z, y, x):
     // 扩展为齐次坐标 (w, z, y, x, 1)
     // 应用旋转矩阵
     [w', z', y', x'] = R * [w, z, y, x]

     // 三线性插值获取新值
     value = trilinearInterpolate(w', z', y', x', originalMatrix)

   // 重采样核心：
   // 1. 取 floor 和 ceil 确定8个最近格点
   // 2. 分别在 w/z/y/x 方向线性插值
   // 3. 最终得到近似原函数的新值
```

**重采样公式（三线性插值）**：

```
已知8个格点值，插值点P(w',z',y',x'):

tx = fract(x'), ty = fract(y'), tz = fract(z'), tw = fract(w')

c000 = lerp(V000, V001, tx)  // 在x方向插值
c001 = lerp(V010, V011, tx)
c010 = lerp(V100, V101, tx)
c011 = lerp(V110, V111, tx)

c00 = lerp(c000, c001, ty)  // 在y方向插值
c01 = lerp(c010, c011, ty)

c0 = lerp(c00, c01, tz)      // 在z方向插值

final = lerp(c0, c1, tw)      // 在w方向插值
```

**限制**：
- 动画过程中禁止切换形状、改变切片值
- 动画进行中按钮全部禁用
- 需优化性能（使用 Float32Array 批量操作）

### 3.3 复位

**前置条件**：存在变换记录且当前状态与初始状态不同

**行为**：
1. 取消正在进行的动画
2. 恢复原始矩阵副本到当前矩阵
3. 重置坐标系到初始状态
4. 重新渲染场景

## 6. 风险与 mitigation

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 4D重采样性能瓶颈 | 中 | 使用 Float32Array 批量操作，预计算插值权重 |
| 动画状态不一致 | 中 | 取消动画前先检查 isAnimating，每次开始前先取消 |
| 内存占用翻倍 | 低 | 变换完成后可释放 originalMatrix（除非需要复位） |
